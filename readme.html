Enthuware JDevPlus Sample Application

This is a sample application that poses similar situation and challenges as the real SCJD assignment. Please remember that this is a SAMPLE application and is not a solution for your real assignment although you can learn how various issues can be dealt with.

The details of the application and the requirements are described in docs\instructionsUC.html

How to execute the program
The entry point into the application is through the class com.unitedcaterers.ApplicationMain.

You can either run this class after extracting the contents of runme.jar or as discribed below:
  1. Let's say you have extracted the contents of the jar file to c:\jdevplus\. You should see the following files in this folder:
    1. javadoc : Contains the generated JavaDoc files.
    2. src  : Contains the source code of the project.
    3. readme.html  : This file.
    4. ucdb.db : The database file.
    5. userguide.txt :  Help on how to use the application.
    6. applicationdesign.html : Important information about the architecture and design of this application.
    7. instructionsuc.html : The instruction file based on which this project was developed. This is similar to the file that you might have received as a part of your SCJD assignement.
    8. version.txt : Contains the version information of the Java compiler and platform used.
    9. runme.jar : Contains the class files with appropriate manifest information.
    10. jdevplus-netbeans : This folder contains netbeans project files. Please see details below.

  2. Run the server. Open up a DOS (command/shell) prompt and go to the above mentioned directory. Execute the following:

    c:\jdevplus\>java -jar runme.jar rmiserver OR c:\jdevplus\>java -jar runme.jar socketserver

    (The server starts RMI registry itself on port 1099, so you don't have to start it.) It will popup a dialog asking for the properties such as database file. Default values will be populated and will work without any change. 
  3. Run the client. Open up a new DOS (command/shell) window and go to same directory. Now execute:

    c:\jdevplus\>java -jar runme.jar rmiclient or c:\jdevplus\>java -jar runme.jar socketclient 

    It will popup a dialog asking for the properties such as hostname and port. Default values will be populated and will work without any change.

    The client can also be run in local mode. In this mode, the Database server and the Client run in the same JVM without any networking logic. You can run it using the following command line:

    c:\jdevplus\>java -jar runme.jar alone

    It will popup a dialog asking for the properties such as database file. Default values will be populated and will work without any change.

Netbeans Project

The folder jdevplus-netbeans contains files specific to NetBeans. It has nothing to do with the the assignment. To use it, open Netbeans IDE, click on File-> Open Project menu, and browse to this folder. Netbeans will recognize this as a valid project and open it. Use "clean and build" to build the project.
Note : The project files assume that your root folder (the folder where you have extracted the contents of rmisocket.zip) to c:\jdevplus folder.

Eclipse Project
The directory structure of the source code is compatible with Ecplise.  Just do New ->Java Project and select "Create Project from Existing Source", to open the project in Eclipse.
